## Prompt
1. **DNS check first:**
   ```bash
   dig +short TXT nuestrascuentitas.com | grep spf
   dig +short TXT resend._domainkey.nuestrascuentitas.com
   ```
   Si no hay SPF/DKIM records: STOP y pedir al user que los agregue en CloudFlare DNS (Resend dashboard los provee). No seguir sin esto.
2. Cargar valores Resend desde Infisical:
   ```bash
   eval $(bash $HOME/.claude/skills/mi-key-cli/scripts/mkey.sh pull bitacora prod --stdout | grep -E '^(RESEND_|ZITADEL_ADMIN_PAT)=')
   ```
3. Enviar config a Zitadel Admin API:
   ```bash
   curl -sS -X PUT -H "Authorization: Bearer $ZITADEL_ADMIN_PAT" -H "Content-Type: application/json" \
     -d "{
       \"senderAddress\": \"$RESEND_SENDER_EMAIL\",
       \"senderName\": \"$RESEND_SENDER_NAME\",
       \"tls\": true,
       \"host\": \"$RESEND_SMTP_HOST:$RESEND_SMTP_PORT\",
       \"user\": \"$RESEND_SMTP_USER\",
       \"password\": \"$RESEND_API_KEY_TESLITA\",
       \"replyToAddress\": \"$RESEND_SENDER_EMAIL\"
     }" \
     https://id.nuestrascuentitas.com/admin/v1/smtp
   ```
4. Disparar test: `curl -sS -X POST -H "Authorization: Bearer $ZITADEL_ADMIN_PAT" -d '{"email":"buhosoft70@gmail.com"}' https://id.nuestrascuentitas.com/admin/v1/smtp/_test`.
5. Esperar al user para confirmar recepcion y pegar headers del mail (Received-SPF, Authentication-Results con DKIM).
6. Persistir reporte. Si DKIM/SPF no son `pass`, abrir sub-issue GH "fix DKIM/SPF nuestrascuentitas.com" y documentar como blocker de Wave A closure.
7. Replicar valores SMTP dentro de Infisical con prefijo ZITADEL_ para trazabilidad explicita:
   ```bash
   mkey set bitacora ZITADEL_SMTP_HOST "$RESEND_SMTP_HOST" --env prod
   mkey set bitacora ZITADEL_SMTP_PORT "$RESEND_SMTP_PORT" --env prod
   mkey set bitacora ZITADEL_SMTP_USER "$RESEND_SMTP_USER" --env prod
   mkey set bitacora ZITADEL_SMTP_PASSWORD "$RESEND_API_KEY_TESLITA" --env prod
   ```
