## Reference
- Zitadel API: POST `/admin/v1/orgs/_search`, POST `/admin/v1/orgs` (create), POST `/management/v1/orgs` (create bajo instancia default)
- T1.2 report — identifica cuales orgs ya existen

## Prompt
1. `PAT=$(bash $HOME/.claude/skills/mi-key-cli/scripts/mkey.sh pull bitacora prod --stdout | grep -E '^ZITADEL_ADMIN_PAT=' | cut -d= -f2-)`.
2. Confirmar lista actual: `curl -sS -X POST -H "Authorization: Bearer $PAT" -H "Content-Type: application/json" -d '{"query":{"limit":100}}' https://id.nuestrascuentitas.com/admin/v1/orgs/_search | jq -r '.result[].name'`.
3. Por cada org requerida que no existe, crearla:
   ```bash
   curl -sS -X POST -H "Authorization: Bearer $PAT" -H "Content-Type: application/json" \
     -d '{"name":"<org-name>"}' \
     https://id.nuestrascuentitas.com/admin/v1/orgs
   ```
   Esto requiere scope `urn:zitadel:iam:org:create`. Si 403, verificar PAT.
4. Re-validar: lista final debe contener las 4.
5. Persistir reporte con IDs y state.

## Execution Procedure
1. Load PAT.
2. List current orgs.
3. Calcular diff vs requeridas.
4. Create las que faltan con curl.
5. Re-list y validar.
6. Commit reporte.
