# Task T2.4: Crear 4 OAuth Clients con redirect placeholders

## Shared Context
**Goal:** dentro de cada Organization, crear un Project y un OAuth Client (public PKCE para SPAs + uno API si aplica) con redirect URIs placeholder que Wave B y siguientes van a actualizar.
**Stack:** Zitadel Management API `/management/v1/projects` + `/management/v1/projects/{id}/apps/oidc`.
**Architecture:** uno por app del ecosistema. Wave B (Bitacora) va a actualizar el client de `bitacora` con redirect URIs reales post-integracion frontend.

## Locked Decisions
- 1 project por org. Nombre project = nombre org para simpleza.
- 1 OIDC App por project, type `OIDC_APP_TYPE_WEB` (PKCE public, para Next.js SPA).
- Agregar 1 API app adicional por project con auth method `client_secret_basic` para backend RS256 validation.
- Redirect URIs placeholder:
  - `https://bitacora.nuestrascuentitas.com/auth/callback`, `/auth/callback/signout` + `http://localhost:3000/auth/callback` (dev)
  - `https://nuestrascuentitas.com/auth/callback` + `http://localhost:3001/auth/callback`
  - `https://multi-tedi.nuestrascuentitas.com/auth/callback` + `http://localhost:3002/auth/callback` (placeholder — dominio real TBD)
  - `https://gastos.nuestrascuentitas.com/auth/callback` + `http://localhost:3003/auth/callback` (placeholder — dominio real TBD)
- Grant types: `authorization_code`, `refresh_token`. Access token type `jwt`. Response types `code`.
- Los client_ids y secrets (para los API apps) se guardan en Infisical con prefijo `ZITADEL_CLIENT_<APP>_ID` y `ZITADEL_CLIENT_<APP>_SECRET`.

## Reference
- POST `/management/v1/projects` (nota: scope mgmt requiere switch de org via header `x-zitadel-orgid`)
- POST `/management/v1/projects/{projectId}/apps/oidc`
- POST `/management/v1/projects/{projectId}/apps/api`
- Header `x-zitadel-orgid: <orgId>` para actuar en el contexto de cada org.

## Prompt
1. Load PAT + org IDs de T2.1 report.
2. Para cada org, repetir flujo:
   ```bash
   ORG_ID=<orgId>
   # Crear project
   PROJECT_ID=$(curl -sS -X POST \
     -H "Authorization: Bearer $ZITADEL_ADMIN_PAT" \
     -H "Content-Type: application/json" \
     -H "x-zitadel-orgid: $ORG_ID" \
     -d '{"name":"<orgName>"}' \
     https://id.nuestrascuentitas.com/management/v1/projects | jq -r '.id')
   # Crear OIDC Web App (PKCE public)
   WEB_APP_RESP=$(curl -sS -X POST \
     -H "Authorization: Bearer $ZITADEL_ADMIN_PAT" \
     -H "Content-Type: application/json" \
     -H "x-zitadel-orgid: $ORG_ID" \
     -d "{
       \"name\":\"<orgName>-web\",
       \"redirectUris\":[\"<redirect1>\",\"<redirect2>\"],
       \"responseTypes\":[\"OIDC_RESPONSE_TYPE_CODE\"],
       \"grantTypes\":[\"OIDC_GRANT_TYPE_AUTHORIZATION_CODE\",\"OIDC_GRANT_TYPE_REFRESH_TOKEN\"],
       \"appType\":\"OIDC_APP_TYPE_WEB\",
       \"authMethodType\":\"OIDC_AUTH_METHOD_TYPE_NONE\",
       \"postLogoutRedirectUris\":[\"<postLogout1>\"],
       \"devMode\":false,
       \"accessTokenType\":\"OIDC_TOKEN_TYPE_BEARER\",
       \"accessTokenRoleAssertion\":false,
       \"idTokenRoleAssertion\":true,
       \"idTokenUserinfoAssertion\":true
     }" \
     "https://id.nuestrascuentitas.com/management/v1/projects/$PROJECT_ID/apps/oidc")
   WEB_CLIENT_ID=$(echo "$WEB_APP_RESP" | jq -r '.clientId')
   # Crear API App (client_credentials para backend)
   API_APP_RESP=$(curl -sS -X POST \
     -H "Authorization: Bearer $ZITADEL_ADMIN_PAT" \
     -H "Content-Type: application/json" \
     -H "x-zitadel-orgid: $ORG_ID" \
     -d "{\"name\":\"<orgName>-api\",\"authMethodType\":\"API_AUTH_METHOD_TYPE_BASIC\"}" \
     "https://id.nuestrascuentitas.com/management/v1/projects/$PROJECT_ID/apps/api")
   API_CLIENT_ID=$(echo "$API_APP_RESP" | jq -r '.clientId')
   API_CLIENT_SECRET=$(echo "$API_APP_RESP" | jq -r '.clientSecret')
   # Persistir en Infisical
   mkey set bitacora ZITADEL_CLIENT_<UPPER_ORG>_WEB_ID "$WEB_CLIENT_ID" --env prod
   mkey set bitacora ZITADEL_CLIENT_<UPPER_ORG>_API_ID "$API_CLIENT_ID" --env prod
   mkey set bitacora ZITADEL_CLIENT_<UPPER_ORG>_API_SECRET "$API_CLIENT_SECRET" --env prod
   mkey set bitacora ZITADEL_PROJECT_<UPPER_ORG>_ID "$PROJECT_ID" --env prod
   ```
3. Tabla de redirects (elegir el correcto por org):
   | Org | Redirects |
   |-----|-----------|
   | bitacora | `https://bitacora.nuestrascuentitas.com/auth/callback`, `http://localhost:3000/auth/callback` |
   | nuestrascuentitas | `https://nuestrascuentitas.com/auth/callback`, `http://localhost:3001/auth/callback` |
   | multi-tedi | `https://multi-tedi.nuestrascuentitas.com/auth/callback`, `http://localhost:3002/auth/callback` |
   | gastos | `https://gastos.nuestrascuentitas.com/auth/callback`, `http://localhost:3003/auth/callback` |
4. Post-logout redirect por app: mismo dominio + `/`.
5. Persistir reporte.
