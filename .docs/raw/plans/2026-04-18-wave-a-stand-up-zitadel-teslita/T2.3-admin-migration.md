## Reference
- POST `/management/v1/users/human/_import`
- PUT `/admin/v1/members/<userId>` (grant IAM Owner)
- POST `/management/v1/users/{userId}/otp/_set` (enroll TOTP)
- Zitadel TOTP: https://zitadel.com/docs/guides/manage/user/reg-create-user#otp

## Prompt
1. `eval $(mkey pull bitacora prod --stdout | grep -E '^ZITADEL_')`.
2. Crear user:
   ```bash
   curl -sS -X POST -H "Authorization: Bearer $ZITADEL_ADMIN_PAT" -H "Content-Type: application/json" \
     -d '{
       "userName": "paz.fgabriel@gmail.com",
       "profile": {"firstName":"Admin","lastName":"Teslita","displayName":"Admin Teslita","preferredLanguage":"es"},
       "email": {"email":"paz.fgabriel@gmail.com","isEmailVerified":false},
       "password":"<tmp-password>",
       "passwordChangeRequired": true
     }' \
     https://id.nuestrascuentitas.com/management/v1/users/human/_import
   ```
   Capturar `userId`.
3. Verificar email: Zitadel envia mail de verificacion (via SMTP wireado en T2.2). El user abre mailbox y click en link.
4. **Pedir al user que inicie sesion y enrolle TOTP**:
   - Abrir `https://id.nuestrascuentitas.com/ui/login`, loguear con paz.fgabriel@gmail.com + tmp-password.
   - Zitadel pide cambiar password. Nueva password guardar en Infisical: `mkey set bitacora ZITADEL_ADMIN_PASSWORD "<new>" --env prod`.
   - Settings > Authenticators > Add Authenticator > Authenticator App. Escanear QR con Google/Microsoft/Aegis Authenticator.
   - Guardar TOTP recovery codes en Infisical: `mkey set bitacora ZITADEL_ADMIN_TOTP_RECOVERY "<codes>" --env prod`.
5. Asignar IAM Owner role:
   ```bash
   curl -sS -X POST -H "Authorization: Bearer $ZITADEL_ADMIN_PAT" -H "Content-Type: application/json" \
     -d "{\"userId\":\"$NEW_ADMIN_USER_ID\",\"roles\":[\"IAM_OWNER\"]}" \
     https://id.nuestrascuentitas.com/admin/v1/members
   ```
6. Validar MFA enrollment:
   ```bash
   curl -sS -X POST -H "Authorization: Bearer $ZITADEL_ADMIN_PAT" -H "Content-Type: application/json" \
     -d "{}" \
     "https://id.nuestrascuentitas.com/management/v1/users/$NEW_ADMIN_USER_ID/auth-factors/_search" | jq '.result[] | select(.otp)'
   ```
7. Desactivar (NO borrar) admin anterior:
   ```bash
   curl -sS -X POST -H "Authorization: Bearer $ZITADEL_ADMIN_PAT" \
     "https://id.nuestrascuentitas.com/management/v1/users/$OLD_ADMIN_USER_ID/_deactivate"
   ```
   Donde `$OLD_ADMIN_USER_ID` salio de T1.2 users list.
8. **Generar PAT nuevo para el admin migrado** y rotar el anterior:
   - Login como admin nuevo en UI.
   - Service Users > Personal Access Tokens > Add. Copiar el nuevo PAT.
   - `mkey set bitacora ZITADEL_ADMIN_PAT "<new-pat>" --env prod` (sobre-escribir el viejo).
   - Revocar PAT anterior desde UI.
9. Persistir reporte con userIds y timestamps.
