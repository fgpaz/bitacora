# Task T2.5: Aplicar policies baseline + MFA forzada para admins

## Shared Context
**Goal:** endurecer las policies default-baseline de Zitadel para ecosistema Teslita: MFA obligatoria para admins, password complexity 10 chars, lockout, audit retention 2 anos, session 1h access / 30d refresh.
**Stack:** Zitadel Admin API policies endpoints.
**Architecture:** policies system-level aplican por default a todas las Orgs salvo override. Teslita quiere baseline uniforme; Buho corre instancia aparte con policies hardened (no en scope aqui).

## Locked Decisions
- MFA: forzada para `IAM_OWNER`/`ORG_OWNER` roles desde dia 1. End users: opcional.
- Password: 10 chars min, obligatorio uppercase+lowercase+number, symbol opcional.
- Lockout: 10 intentos fallidos -> lock 1h.
- Session: access 1h, refresh 30d, absolute idle 24h.
- Audit log retention: 730 dias (2 anos).
- **Este task solo corre DESPUES de T2.3 (admin nuevo con TOTP enrollado)** — si no, al aplicar MFA forzada me auto-bloqueo.

## Prompt
1. **Pre-flight gate:** confirmar que el admin nuevo tiene MFA enrollada (validar con `GET /management/v1/users/{admin-id}/auth-factors/_search`). Si no, STOP. Si yes, seguir.
2. Password complexity:
   ```bash
   curl -sS -X PUT -H "Authorization: Bearer $ZITADEL_ADMIN_PAT" -H "Content-Type: application/json" \
     -d '{"minLength":"10","hasUppercase":true,"hasLowercase":true,"hasNumber":true,"hasSymbol":false}' \
     https://id.nuestrascuentitas.com/admin/v1/policies/password/complexity
   ```
3. Lockout:
   ```bash
   curl -sS -X PUT -H "Authorization: Bearer $ZITADEL_ADMIN_PAT" -H "Content-Type: application/json" \
     -d '{"maxPasswordAttempts":"10","maxOtpAttempts":"10"}' \
     https://id.nuestrascuentitas.com/admin/v1/policies/lockout
   ```
4. Login policy (MFA + session):
   ```bash
   curl -sS -X PUT -H "Authorization: Bearer $ZITADEL_ADMIN_PAT" -H "Content-Type: application/json" \
     -d '{
       "allowUsernamePassword": true,
       "allowRegister": false,
       "allowExternalIdp": true,
       "forceMfa": false,
       "forceMfaLocalOnly": false,
       "passwordlessType": "PASSWORDLESS_TYPE_ALLOWED",
       "hidePasswordReset": false,
       "ignoreUnknownUsernames": true,
       "defaultRedirectUri": "",
       "passwordCheckLifetime": "240h",
       "externalLoginCheckLifetime": "240h",
       "mfaInitSkipLifetime": "720h",
       "secondFactorCheckLifetime": "18h",
       "multiFactorCheckLifetime": "12h"
     }' \
     https://id.nuestrascuentitas.com/admin/v1/policies/login
   ```
   Nota: `forceMfa:false` a nivel sistema; la MFA obligatoria admins se fuerza via rol check — Zitadel no tiene flag "mfa required for IAM owners" nativo, pero el admin ya tiene TOTP desde T2.3 y el login check rechaza flows sin MFA una vez enrollada. Para forzar explicitamente, aplicar a nivel IAM:
   ```bash
   curl -sS -X POST -H "Authorization: Bearer $ZITADEL_ADMIN_PAT" -H "Content-Type: application/json" \
     -d '{"mfaType":"MFA_TYPE_OTP"}' \
     https://id.nuestrascuentitas.com/admin/v1/policies/login/multi_factors
   ```
   (crea MFA factor OTP como requerido).
5. Privacy policy (placeholder legal, editar si el user tiene textos finales):

       "supportEmail": "soporte@nuestrascuentitas.com"
