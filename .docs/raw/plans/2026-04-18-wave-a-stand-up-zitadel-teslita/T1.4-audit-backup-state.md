# Task T1.4: Audit backup state VPS + Postgres Zitadel volume

## Shared Context
**Goal:** descubrir si existe backup job para la Postgres Zitadel, nombre exacto del volumen Docker, espacio libre en VPS, crons existentes.
**Stack:** Dokploy API + SSH VPS (via sshr si disponible, sino via Dokploy container shell).
**Architecture:** Backup decidido en brainstorming: **docker volume snapshot + offsite**. Para disenarlo hay que conocer el volumen real, el espacio disponible, y si ya hay algo corriendo.

- Backup strategy lockeada: snapshot del volumen Docker de la Postgres + offsite destination (definir destino en T3.1, por ahora solo auditar).
- Read-only. Sin mutar cron ni volumenes.

## Prompt
1. **Volumen exacto:** del reporte T1.1, extraer el volumen declarado del Postgres Zitadel (algo como `dokploy_postgres-xxx_data` o similar).
2. Si hay sshr disponible (`which sshr`), ejecutar: `sshr exec --host turismo --cmd "docker volume inspect <volumen>" > /tmp/vol.json`. Sino, usar `dkp.sh` para obtener container-shell-access:
   - `dkp.sh GET application.one applicationId=<postgresAppId>` para confirmar volumen.
   - Alternative: pedir al user que corra el comando via SSH y pegue el output.
3. **Espacio VPS:** `sshr exec --host turismo --cmd "df -h / && docker system df"` -> persistir output.
4. **Crons:** `sshr exec --host turismo --cmd "crontab -l 2>/dev/null; ls /etc/cron.* 2>/dev/null"` -> listar jobs existentes. Buscar si hay algo relacionado con backups Zitadel.
5. **Backups existentes:** `sshr exec --host turismo --cmd "ls -la /var/backups 2>/dev/null; ls -la /home/dokploy/backups 2>/dev/null"` -> ver si Dokploy o algo mas ya hace backups.
6. **Candidate offsite destinations** (sin commit, solo investigar):
   - Backblaze B2: pricing $0.006/GB-mes, requiere B2 account + bucket + app key.
   - Cloudflare R2: $0.015/GB-mes, S3-compat, free egress.
   - AWS S3 region `sa-east-1`: mayor latencia, $0.023/GB-mes.
   - rsync a desktop del user via Tailscale: gratis pero depende de up-time del desktop.
   - **Otro VPS Hetzner Storage Box:** $3.81/mes 1TB, 10 hops simple.
   Reportar trade-offs para que user decida en T3.1.
7. Persistir reporte con skeleton.

## Execution Procedure
1. Cargar volumen Postgres Zitadel desde T1.1.
2. Ejecutar comandos ssh (o pedir al user si sshr no esta). Capturar outputs.
3. Persistir reporte + commit.
