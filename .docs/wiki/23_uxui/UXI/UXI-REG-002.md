# UXI-REG-002 — Intención de Registro de factores diarios vía web

## Propósito

Este documento fija la intención emocional y operativa del slice `REG-002`.

No describe todavía la journey completa ni el detalle de una pantalla. Su función es decidir cómo debe sentirse el caso para que `UJ`, `VOICE` y `UXS` no improvisen tono, ritmo o confianza.

## Sensación deseada

- compacto
- ordenado
- comprensible
- sin juicio
- útil

La intención de este caso es que la experiencia se sienta como `check-in compacto`.

## Anti-sensación

La anti-sensación principal a evitar es:

**una encuesta clínica larga y moralizante.**

## Postura de confianza

la confianza se sostiene mostrando solo lo necesario y evitando tono evaluativo.

## Goal del actor

registrar factores del día sin sentir que completa un examen.

## Fricción aceptable

- mostrar un bloque condicional solo cuando corresponde
- feedback breve al guardar

## Fricción indebida

- mostrar todos los campos de una vez
- agregar texto clínico explicativo
- pedir precisión innecesaria

## Defaults del caso

- hereda la base del sistema: casi nula fricción, seguridad implícita, simpleza radical, paciente primero cuando el actor principal es la persona usuaria, silencio útil fuera de los momentos sensibles, explicitud alta solo cuando cambian acceso, datos o consentimiento;
- la sensación dominante del slice es `compacto`, pero sin sacrificar claridad sensible;
- la voz y la interfaz deben sostener una sola dirección dominante por paso.

## Criterio de validación rápida

Este `UXI` está bien calibrado si el caso se percibe como:

- compacto
- ordenado
- comprensible
- sin juicio
- útil

Y está mal calibrado si se percibe como:

- una encuesta clínica larga y moralizante
- una experiencia que pide más esfuerzo que el valor que devuelve.
