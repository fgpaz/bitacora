# UXI-VIN-003 — Intención de Revocación de vínculo por paciente

## Propósito

Este documento fija la intención emocional y operativa del slice `VIN-003`.

No describe todavía la journey completa ni el detalle de una pantalla. Su función es decidir cómo debe sentirse el caso para que `UJ`, `VOICE` y `UXS` no improvisen tono, ritmo o confianza.

## Sensación deseada

- sereno
- firme
- claro
- bajo control

La intención de este caso es que la experiencia se sienta como `corte claro`.

## Anti-sensación

La anti-sensación principal a evitar es:

**romper algo sin entender impacto.**

## Postura de confianza

la interfaz debe mostrar que cortar el vínculo es una capacidad legítima del paciente.

## Goal del actor

suspender el acceso de un profesional específico sin ruido innecesario.

## Fricción aceptable

- una confirmación explícita
- explicar que el acceso cae de inmediato

## Fricción indebida

- copy culpabilizante
- igualar vínculo con consentimiento
- mostrar consecuencias vagas

## Defaults del caso

- hereda la base del sistema: casi nula fricción, seguridad implícita, simpleza radical, paciente primero cuando el actor principal es la persona usuaria, silencio útil fuera de los momentos sensibles, explicitud alta solo cuando cambian acceso, datos o consentimiento;
- la sensación dominante del slice es `sereno`, pero sin sacrificar claridad sensible;
- la voz y la interfaz deben sostener una sola dirección dominante por paso.

## Criterio de validación rápida

Este `UXI` está bien calibrado si el caso se percibe como:

- sereno
- firme
- claro
- bajo control

Y está mal calibrado si se percibe como:

- romper algo sin entender impacto
- una experiencia que pide más esfuerzo que el valor que devuelve.
