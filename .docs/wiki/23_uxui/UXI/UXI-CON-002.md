# UXI-CON-002 — Intención de Revocación de consentimiento

## Propósito

Este documento fija la intención emocional y operativa del slice `CON-002`.

No describe todavía la journey completa ni el detalle de una pantalla. Su función es decidir cómo debe sentirse el caso para que `UJ`, `VOICE` y `UXS` no improvisen tono, ritmo o confianza.

## Sensación deseada

- serio
- sereno
- claro
- no punitivo

La intención de este caso es que la experiencia se sienta como `cierre responsable`.

## Anti-sensación

La anti-sensación principal a evitar es:

**una amenaza o castigo disfrazado de configuración.**

## Postura de confianza

la seguridad del sistema también consiste en permitir salir con claridad.

## Goal del actor

revocar consentimiento entendiendo el efecto sobre registro y accesos.

## Fricción aceptable

- una pausa deliberada de mayor intensidad que otros formularios
- confirmación explícita de impacto

## Fricción indebida

- copy amenazante
- legalese sin traducción humana
- ocultar la cascada a vínculos

## Defaults del caso

- hereda la base del sistema: casi nula fricción, seguridad implícita, simpleza radical, paciente primero cuando el actor principal es la persona usuaria, silencio útil fuera de los momentos sensibles, explicitud alta solo cuando cambian acceso, datos o consentimiento;
- la sensación dominante del slice es `serio`, pero sin sacrificar claridad sensible;
- la voz y la interfaz deben sostener una sola dirección dominante por paso.

## Criterio de validación rápida

Este `UXI` está bien calibrado si el caso se percibe como:

- serio
- sereno
- claro
- no punitivo

Y está mal calibrado si se percibe como:

- una amenaza o castigo disfrazado de configuración
- una experiencia que pide más esfuerzo que el valor que devuelve.
