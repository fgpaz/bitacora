# 19 — Índice UJ por slice

## Propósito

Este documento funciona como índice/puente de la capa `UJ`.

Las journeys por caso viven en `23_uxui/UJ/*` y modelan tareas completas, no pantallas aisladas.

## Regla operativa

- cada slice visible debe tener un `UJ-<SLICE>.md` propio;
- la journey debe nombrar ritmo, errores relevantes y paso crítico;
- si un caso cambia de actor o de conflicto central, debe abrir un slice nuevo.
